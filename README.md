# VAMP Docs

This repository contains the public-facing documentation site for VAMP.

It is built with [Astro](https://astro.build/) and [Starlight](https://starlight.astro.build/) and currently publishes the VAMP homepage plus the research and proposal material under the `Research` section.

## Local development

Install dependencies and start the site locally:

> [!IMPORTANT]
> We use [Yarn](https://yarnpkg.com/) to manage dependencies. If you don't have it installed, you can get it from the link above or use `npm` with equivalent commands. Please do not try to mix package managers.

```bash
yarn install
yarn dev
```

Useful commands:

| Command | Purpose |
| --- | --- |
| `yarn dev` | Start the local development server |
| `yarn build` | Build the production site into `dist/` |
| `yarn preview` | Preview the production build locally |
| `yarn astro ...` | Run Astro CLI commands |

## Contributing

Keep changes small and focused on the content or site behavior you are touching.

- Write normal docs pages in `src/content/docs/`.
- The homepage lives at `src/content/docs/index.mdx`.
- Research content lives in `src/content/docs/research/` and is added to the sidebar automatically.
- Prefer `.md` for standard pages and use `.mdx` only when you need components or richer layout control.
- Put shared styling in `src/styles/site.css`.
- Put reusable UI overrides or additions in `src/components/`.
- Run `yarn build` before shipping meaningful changes.

## Project structure

The main files and folders contributors will usually work with are:

```text
.
├── public/                  # Static assets
├── src/
│   ├── assets/              # Images and local assets used by the site
│   ├── components/          # Custom Astro/Starlight components
│   ├── content/
│   │   └── docs/            # Documentation content
│   │       ├── index.mdx    # Homepage
│   │       └── research/    # Research pages
│   ├── styles/
│   │   └── site.css         # Site-wide custom styles
│   └── content.config.ts    # Starlight content collection config
├── astro.config.mjs         # Astro + Starlight site configuration
└── package.json
```

This repo currently uses a single Starlight docs collection and an autogenerated `Research` sidebar group. New pages added under `src/content/docs/research/` will show up in navigation without extra sidebar wiring.
